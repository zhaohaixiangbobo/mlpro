# 数据预处理参数与串联优化计划

根据您的要求，我制定了以下计划来优化列选择功能并修复预处理模块的串联问题：

## 1. 优化列选择 UI (`MissingValueForm.tsx`, `StandardizationForm.tsx`)

*   **多选与复选框**: 废弃 `Select` 组件，改用 `Checkbox.Group` 结合自定义布局来实现列选择。
*   **全选/取消**:
    *   在列表顶部添加“全选”复选框。逻辑：当所有列都被选中时自动勾选；点击时切换所有列的选中状态。
    *   添加“清除选择”按钮/链接，一键清空。
*   **状态反馈**: 如果未选择任何列，显示提示信息（如警告图标或文字说明）。

## 2. 修复模块串联与数据流 (`Workflow.tsx`, `PropertyPanel.tsx`)

*   **问题根源**: 当前 `PropertyPanel` 仅依赖直接相连的 `inputNode` 获取文件名。如果中间插入了预处理节点（它没有 `filename`），后续节点就无法获取列信息。
*   **解决方案**:
    *   在 `Workflow.tsx` 中实现递归查找函数 `findRootDataNode(node)`，它将沿着连接线向上追溯，直到找到源头的 `DataNode`。
    *   将这个 `rootDataNode` 传递给 `PropertyPanel`。
    *   这样，无论预处理节点串联多少级，它们都能从根节点获取原始数据文件的列信息。
    *   (注：目前假设预处理步骤（如填充、标准化）不改变列名或列数，因此复用原始列信息是安全的且满足需求的)。

## 3. 执行步骤

1.  **修改 `Workflow.tsx`**:
    *   实现 `findRootDataNode` 逻辑。
    *   将计算出的 `rootNode` 传递给 `PropertyPanel`。
2.  **修改 `PropertyPanel.tsx`**:
    *   接收 `rootNode` 属性。
    *   使用 `rootNode.data.filename` 来获取列列表，替代原本的 `inputNode` 逻辑。
3.  **重构 `MissingValueForm.tsx`**:
    *   实现全选、反选、多选 Checkbox 逻辑。
    *   美化 UI，区分已选/未选。
4.  **重构 `StandardizationForm.tsx`**:
    *   同样实现全选、反选、多选 Checkbox 逻辑。

此计划将确保用户体验流畅，并且预处理模块可以自由串联而不丢失列配置能力。
