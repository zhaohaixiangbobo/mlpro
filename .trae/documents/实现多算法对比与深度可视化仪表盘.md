# 重构可视化结果 (Dashboard) 页面

本计划将全面升级 `Dashboard.tsx` 页面，实现工作流结果的深度可视化与多算法对比。

## 1. 核心功能设计

### 1.1 工作流加载与管理
*   **顶部工具栏**: 
    *   下拉选择框：列出所有已保存的工作流。
    *   **默认行为**: 页面加载时自动请求工作流列表，并加载 `updated_at` 最新的一个。
*   **数据解析**:
    *   从工作流 JSON 中提取所有 `algoNode`。
    *   按 `category`（分类、回归、聚类）对算法进行分组。
    *   仅处理 `status === '成功'` 且包含 `result` 数据的节点。

### 1.2 多维可视化展现

#### 情况 A：同类型多算法对比 (核心功能)
*   **对比表格**: 展示各算法的关键指标（Accuracy, Precision, Recall, F1-Score 或 MSE, R2, MAE）。
*   **雷达图 (Radar Chart)**: 将各项指标归一化后，在同一个坐标系下展示不同算法的性能轮廓（最直观的优劣对比）。
*   **排行条形图**: 针对核心指标（如 Accuracy 或 R2）进行排序展示。

#### 情况 B：分类算法详情
*   **模型参数列表**: 展示训练时的具体参数。
*   **混淆矩阵 (Heatmap)**: 如果 `result` 中包含 `confusion_matrix`，使用热力图展示。
*   **详细报告 (Report)**: 以表格形式展示每个类别的 Precision/Recall/F1。

#### 情况 C：聚类算法可视化
*   **指标概览**: 轮廓系数 (Silhouette Score)、Calinski-Harabasz Score 等。
*   **降维散点图 (PCA)**: 
    *   **后端配合**: 需要新增一个 `/api/data/reduce_dimension` 接口，执行 PCA 降维。
    *   **前端展示**: 将降维后的数据点按聚类结果（簇 ID）着色展示。

## 2. 技术实施路径

### 2.1 后端增强 (`api/data.py` & `ml_service.py`)
*   **新增接口 `/api/data/pca`**:
    *   接收文件名、特征列、聚类结果。
    *   返回降维后的二维坐标 (x, y) 和对应的标签。

### 2.2 前端开发 (`Dashboard.tsx`)
*   **图表库选择**: 使用 `ant-design/charts` (基于 G2Plot) 或 `recharts`。鉴于本项目已使用 Ant Design，优先推荐 **`@ant-design/plots`**。
*   **页面布局**:
    *   `Layout.Header`: 工作流切换。
    *   `Tabs`: 区分“分类对比”、“回归对比”、“聚类分析”。
    *   `Row/Col` 栅格布局：放置各种图表卡片。

## 3. 执行步骤
1.  **Backend**: 实现 PCA 降维接口，支持聚类结果的可视化。
2.  **Frontend**: 安装图表库依赖。
3.  **Frontend**: 编写 `Dashboard.tsx` 逻辑：
    *   实现自动加载最新工作流。
    *   实现数据分组与指标提取。
    *   编写各种可视化组件（对比表、雷达图、条形图）。
4.  **Verification**: 保存一个包含多个分类模型的工作流，验证 Dashboard 的对比效果。
