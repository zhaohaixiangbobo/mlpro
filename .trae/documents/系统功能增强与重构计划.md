# 系统功能增强与重构计划

## 问题分析与修复策略

1.  **文件列表显示空白问题**:
    *   **问题**: `ModelParamsForm.tsx` 中的下拉框 `Option` 使用 `file.name` 作为 key 和 value，但后端 `/data/list` 返回的是简单的字符串数组 `["file1.csv", "file2.csv"]`，而不是对象数组 `[{name: "file1.csv"}, ...]`。因此 `file.name` 为 `undefined`，导致列表项空白。
    *   **修复**: 修改 `ModelParamsForm.tsx`，将 `file.name` 改为直接使用 `file`（即字符串本身）。

2.  **模型评估节点合并与重构**:
    *   **问题**: 侧边栏有三个独立的评估节点，不够通用。且评估逻辑目前混杂在 `run_algorithm` 中，不够清晰，且缺少针对不同任务类型的详细指标。
    *   **修复 (Sidebar)**: 在 `Sidebar.tsx` 中，将三个评估节点合并为一个通用的“模型评估”节点。
    *   **修复 (Backend)**: 在 `ml_service.py` 中，将评估逻辑提取为 `_evaluate_model` 私有方法。根据任务类型（分类、回归、聚类）返回不同的指标集（Accuracy, Confusion Matrix, Report / MSE, RMSE, MAE, R2 / Silhouette, Inertia）。
    *   **修复 (Frontend)**: 更新 `EvalNode.tsx`，使其能够根据 `data.result` 中的字段动态渲染不同的评估指标。

3.  **训练集/测试集 Loss 情况**:
    *   **策略**: 对于支持 `loss_curve_` 的模型（如 GBDT, MLP），后端应返回 loss 曲线数据。对于其他模型（如随机森林），由于没有迭代 loss，暂无法提供。可以在后续版本中通过 `learning_curve` 函数实现通用的学习曲线，但本次先聚焦于基础评估指标的完善。本次将优先完善回归任务的 MSE/MAE/RMSE 和分类任务的详细报告。

## 实施步骤

### 1. 修复前端文件列表 (`frontend/src/components/params/ModelParamsForm.tsx`)
*   修改 `fileList.map` 逻辑，直接使用字符串项。

### 2. 重构侧边栏 (`frontend/src/components/Sidebar.tsx`)
*   移除 `基础评估`, `混淆矩阵`, `ROC曲线`。
*   添加单一的 `模型评估` 节点。

### 3. 重构后端评估逻辑 (`backend/app/services/ml_service.py`)
*   新增 `_evaluate_model` 方法。
*   **分类**: 返回 Accuracy, Confusion Matrix, Classification Report。
*   **回归**: 返回 MSE, RMSE (`np.sqrt(mse)`), MAE (`mean_absolute_error`), R2。
*   **聚类**: 返回 Silhouette Score, Inertia (KMeans only), Cluster Counts。
*   在 `run_algorithm` 中调用该方法。

### 4. 更新评估节点 UI (`frontend/src/components/nodes/EvalNode.tsx`)
*   重写渲染逻辑。
*   如果是分类结果（有 `accuracy`），显示分类指标。
*   如果是回归结果（有 `mse`），显示回归指标。
*   如果是聚类结果（有 `silhouette_score`），显示聚类指标。

## 验证计划
1.  **文件列表**: 打开算法节点，确认测试集文件下拉框显示文件名。
2.  **评估节点**: 拖入新的“模型评估”节点，运行分类任务，检查是否显示准确率和混淆矩阵。
3.  **回归指标**: 运行线性回归任务，检查评估节点是否显示 MSE, RMSE, MAE, R2。
