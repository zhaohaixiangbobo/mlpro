# 修复工作流页面问题及功能增强计划

针对您提出的三个问题，我制定了以下修复和增强计划：

## 1. 修复 "清空" 按钮显示问题
- **问题分析**: 之前的按钮使用了默认样式配合 `danger` 属性，可能导致仅显示红色边框和文字，不够显眼。
- **解决方案**: 将 "清空" 按钮样式修改为 `type="primary" danger`，使其显示为**实心红色按钮**，提高可见性。

## 2. 修复删除工作流功能
- **问题分析**: 用户反馈点击确认后未删除，可能是 URL 编码问题（如果名称包含中文）或请求未正确发送。
- **解决方案**: 
    - 在前端 API 调用中对工作流名称进行 `encodeURIComponent` 编码，确保中文名称能被正确传输。
    - 检查 `Popconfirm` 的 `onConfirm` 回调绑定。
    - 增加更详细的错误日志以便排查。

## 3. 限制数据输入节点数量
- **问题分析**: 目前画布未限制数据节点数量，导致逻辑混乱。
- **解决方案**: 
    - 在 `onDrop`（拖拽放置）事件中增加逻辑判断。
    - 如果画布中已存在 `dataNode` 类型的节点，阻止放置新节点并弹出警告提示："画布中只能存在一个数据输入节点"。

- `d:\2-code\mlpro\frontend\src\pages\Workflow.tsx`: 修改按钮样式和 `onDrop` 逻辑。
- `d:\2-code\mlpro\frontend\src\services\api.ts`: 优化 `deleteWorkflow` 的 URL 编码。
