# 系统功能增强与修复计划

## 问题分析与修复策略

1. **数据划分保存路径错误 (Backend)**:

   * **问题**: `data_service.py` 中的 `save_split` 函数使用了未定义的全局变量 `DATA_DIR`。

   * **修复**: 将 `DATA_DIR` 替换为 `self.data_dir`。

2. **算法参数界面优化 (Frontend)**:

   * **解释功能**: 为 `ModelParamsForm` 中的每个参数添加 `tooltip`，提供鼠标悬停解释。

   * **重置按钮位置**: 将重置按钮从底部移至标题栏右侧，提升可用性。

3. **随机种子参数 (Full Stack)**:

   * **前端**: 在所有算法的参数配置中添加 `random_state` 输入框（默认 42）。

   * **后端**: 修改 `MLService.py`，从参数中读取 `random_state`，并将其应用到 `train_test_split` 和模型初始化中，确保结果可复现。

4. **显式测试集支持 (Full Stack)**:

   * **前端**: 在 `ModelParamsForm` 中增加一个 `test_file` 输入框（可选）。

   * **后端**: 修改 `MLService.py` 的 `run_algorithm` 逻辑。如果 `params` 中包含 `test_file`，则加载该文件作为测试集（跳过 `train_test_split`），否则继续使用默认的自动切分逻辑。

## 实施步骤

### 1. 修复后端数据服务

* **文件**: `backend/app/services/data_service.py`

* **任务**: 修正 `save_split` 中的路径构建代码。

### 2. 增强后端 ML 服务

* **文件**: `backend/app/services/ml_service.py`

* **任务**:

  * 在 `run_algorithm` 中支持 `test_file` 参数。

  * 实现显式测试集加载逻辑（需对测试集应用相同的预处理）。

  * 支持 `random_state` 参数传递给切分函数和模型。

### 3. 优化前端参数面板

* **文件**: `frontend/src/components/params/ModelParamsForm.tsx`

* **任务**:

  * 重构 UI 布局，将重置按钮移至顶部。

  * 为所有字段添加 `tooltip` 解释。

  * 为所有模型添加公共参数字段：`random_state` 和 `test_file`。

  * 更新 `defaultParams` 以包含 `random_state: 42`。

## 验证计划

1. **保存功能**: 运行数据划分功能，检查生成的 CSV 文件是否正确保存在 `data` 目录下。
2. **UI 交互**: 悬停参数查看解释；点击顶部重置按钮确认功能正常。
3. **可复现性**: 使用相同的 `random_state` 运行两次算法，确认结果（准确率、混淆矩阵）完全一致。
4. **自定义测试集**: 输入一个已存在的测试文件名，运行算法，确认后端日志或结果表明使用了该独立文件进行评估。
