# 增加预测功能界面 (Prediction Tab)

本计划将新增一个独立的“模型预测”页面，允许用户选择已训练成功的算法模型，对新数据进行预测并下载结果。

## 1. 后端修改 (Backend)

### 1.1 模型持久化 (`ml_service.py`)
*   **目标**: 在算法运行成功后保存模型，以便后续预测使用。
*   **修改**:
    *   引入 `joblib` 库。
    *   新增 `save_model(model, node_id)` 方法，将模型保存为 `.joblib` 文件。
    *   修改 `run_algorithm` 方法，接收 `node_id` 参数，并在训练成功后调用 `save_model`。
    *   新增 `predict(node_id, data_file, preprocessing)` 方法：
        *   加载保存的模型。
        *   加载预测数据文件。
        *   执行预处理步骤（与训练时一致）。
        *   调用 `model.predict`。
        *   将结果保存为 CSV 文件。
        *   返回预览数据（前几行）和下载文件名。

### 1.2 API 接口更新 (`workflow.py`)
*   **修改 `RunRequest`**: 增加 `node_id` 字段 (Optional)。
*   **新增 `PredictRequest`**: 包含 `node_id`, `data_file`, `preprocessing` 等字段。
*   **新增 `/predict` 接口**: 处理预测请求。
*   **新增 `/download/prediction/{filename}` 接口**: 用于下载预测结果。

## 2. 前端修改 (Frontend)

### 2.1 API 服务 (`api.ts`)
*   新增 `predict(payload)` 方法。
*   新增 `downloadPrediction(filename)` 方法。

### 2.2 工作流页面 (`Workflow.tsx`)
*   **修改**: 在点击“运行”时，将当前节点的 `id` 传递给后端，确保模型文件与节点一一对应。

### 2.3 新增预测页面 (`pages/Prediction.tsx`)
*   **功能布局**:
    1.  **选择工作流**: 下拉框列出所有工作流。
    2.  **选择模型**: 选中工作流后，自动解析 JSON，筛选出状态为“成功”的算法节点供用户选择。
    3.  **上传文件**: 上传用于预测的新数据文件 (CSV/Excel)。
    4.  **执行预测**: 点击按钮调用后端接口。
    5.  **结果展示**: 表格展示预测结果的前 5-10 行。
    6.  **下载结果**: 提供 CSV 下载按钮。

### 2.4 路由与导航 (`App.tsx`, `Sidebar.tsx`)
*   **Sidebar**: 增加“模型预测”菜单项。
*   **Router**: 注册 `/prediction` 路由。

## 3. 执行步骤
1.  **Backend**: 更新 `ml_service.py` 和 `workflow.py`，实现模型保存与预测逻辑。
2.  **Frontend**: 更新 `api.ts` 和 `Workflow.tsx`。
3.  **Frontend**: 创建 `Prediction.tsx` 并配置路由。
4.  **验证**: 运行一个工作流，确保模型被保存；切换到预测页面，选择该模型并上传数据进行预测。
