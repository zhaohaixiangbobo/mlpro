# 系统功能增强与修复计划

## 问题分析与修复策略

1.  **标签编码 (Label Encoding)**:
    *   **问题**: 部分分类算法（如 XGBoost）需要数值型的标签，直接使用字符串标签会导致报错。
    *   **解决**: 在 `ml_service.py` 中引入 `LabelEncoder`。在分类任务的数据准备阶段，对标签列 `y` 进行 `fit_transform` 转换为整数编码。

2.  **测试集文件选择 (File Selection)**:
    *   **问题**: 算法节点的 `test_file` 参数目前是手动输入的文本框，容易出错且不便。
    *   **解决**: 在 `ModelParamsForm.tsx` 中引入 `api.listFiles()`。将输入框改为 `Select` 下拉组件，自动列出数据管理中已上传的文件供选择。
    *   **逻辑调整**: 仅当算法类型不是“聚类算法”（K-Means, DBSCAN）时，才显示此选项。

3.  **用户反馈入口 (Feedback Menu)**:
    *   **问题**: 用户希望在退出登录下方增加问题反馈入口。
    *   **解决**: 修改 `MainLayout.tsx` 中的 `userMenu`，添加一项“问题反馈”。点击后弹出一个 `Modal` 对话框，显示联系人信息（赵海翔 166xxx8957）。

## 实施步骤

### 1. 后端修改 (`backend/app/services/ml_service.py`)
*   **导入**: `from sklearn.preprocessing import LabelEncoder`
*   **逻辑**:
    *   在分类任务分支中，初始化 `le = LabelEncoder()`。
    *   对 `y` 执行 `y = le.fit_transform(y)`。
    *   如果有显式测试集，对 `y_test` 执行 `y_test = le.transform(y_test)`（注意处理未见过的标签异常）。

### 2. 前端修改 (`frontend/src/components/params/ModelParamsForm.tsx`)
*   **状态管理**: 添加 `fileList` state。
*   **副作用**: `useEffect` 中调用 `listFiles` 获取文件列表。
*   **UI 更新**: 将 `renderCommonFields` 中的 `test_file` 输入框改为 `Select`。
*   **条件渲染**: 在聚类算法（K-Means, DBSCAN）中隐藏 `test_file` 字段。

### 3. 布局修改 (`frontend/src/components/MainLayout.tsx`)
*   **菜单项**: 在 `userMenu` items 数组中添加 `key: 'feedback'` 项。
*   **交互**: 添加 `Modal` 状态控制，点击反馈菜单时显示弹窗，弹窗内容为指定的联系方式。

## 验证计划
1.  **分类任务**: 使用带有字符串标签（如 Iris 数据集的 species 列）的数据集运行 XGBoost，确认不再报错。
2.  **文件选择**: 打开算法节点参数面板，确认“测试集文件”可以下拉选择已上传的文件。
3.  **反馈弹窗**: 点击右上角头像 -> 问题反馈，确认弹出包含正确联系人信息的对话框。
