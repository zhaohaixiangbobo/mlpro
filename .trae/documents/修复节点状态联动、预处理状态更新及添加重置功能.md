# 修复节点状态联动、预处理状态更新及添加重置功能

## 问题分析
1.  **节点参数联动问题**:
    *   **原因**: `ModelParamsForm` 中的 `useEffect` 仅依赖 `modelType`。当切换到同类型的另一个节点时（例如两个逻辑回归节点），`modelType` 未变，导致 `form.resetFields()` 不执行，表单保留了上一个节点的参数值。这让用户误以为修改的是共享状态，实际上是显示未更新。
    *   **解决**: 将 `useEffect` 的依赖项改为 `[data.id, modelType, form]`，确保每次选中不同节点时表单都会重置为该节点的实际数据。

2.  **预处理节点状态未更新**:
    *   **原因**: `handleRun` 函数在更新节点状态时，只查找了 `results` 中的 `nodeId` (即模型节点) 和其下游的评估节点。上游的预处理节点（虽然在 `traceBackPath` 中被收集）没有被包含在状态更新的逻辑中。
    *   **解决**: 修改 `handleRun`，在 `results` 对象中不仅返回模型节点的 ID，还返回该路径上所有预处理节点的 ID (`pathNodeIds`)。然后在 `setNodes` 遍历时，检查节点 ID 是否在 `pathNodeIds` 中，如果是则更新状态为“成功”。

3.  **参数无法重置**:
    *   **原因**: UI 上缺少重置按钮。
    *   **解决**: 在 `ModelParamsForm` 底部添加一个“重置”按钮，调用 Ant Design Form 的 `resetFields()` 方法，并触发 `onChange` 将数据恢复为默认值。

## 实施计划

### 1. 修复 `ModelParamsForm.tsx` (参数联动与重置)
*   **修改 `useEffect`**: 添加 `data.id` 到依赖数组。
    ```typescript
    useEffect(() => {
        form.resetFields();
    }, [data.id, modelType, form]); // 关键修复
    ```
*   **添加重置按钮**: 在提交按钮旁添加 `<Button onClick={handleReset}>重置</Button>`。
*   **实现 `handleReset`**: 调用 `form.resetFields()` 并触发 `onChange` 更新节点数据。

### 2. 修复 `Workflow.tsx` (预处理节点状态)
*   **修改 `runPromises` 返回结构**:
    ```typescript
    return {
        nodeId: algoNode.id,
        pathNodeIds: preprocessingNodes.map(n => n.id), // 新增：记录路径上的所有节点
        success: true,
        result: response.data.result
    };
    ```
*   **更新状态逻辑**:
    在 `setNodes` 中：
    ```typescript
    // 收集所有成功路径上的节点 ID
    const successfulPathIds = new Set<string>();
    results.forEach(r => {
        if (r && r.success) {
            r.pathNodeIds.forEach(id => successfulPathIds.add(id));
        }
    });

    // ... inside map ...
    if (successfulPathIds.has(node.id)) {
        return { ...node, data: { ...node.data, status: '成功' } };
    }
    ```

### 3. 验证
*   **测试场景 1**: 创建两个逻辑回归节点，修改 A 的参数，点击 B，确认 B 的参数面板显示默认值（或 B 自己的值），而不是 A 的值。
*   **测试场景 2**: 运行流程 `Data -> MissingValue -> LogicRegression`，确认运行成功后，`MissingValue` 节点状态变为“成功”。
*   **测试场景 3**: 修改参数后点击“重置”，确认参数恢复默认。

该计划将全面解决您提出的三个问题。
