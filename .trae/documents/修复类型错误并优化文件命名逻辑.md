# 修复 Prediction.tsx 类型错误与逻辑优化

**问题原因**:
1. **类型错误**: 在 `Steps` 组件的 `items` 属性中，`Select` 的 `onChange` 事件直接绑定了 `handleWorkflowChange`。然而，`Select` 的 `onChange` 传递的是 `(value: string)`，而 `handleWorkflowChange` 的签名是 `(value: string, restoreAlgoId?: string) => Promise<void>`。虽然 JS 运行时兼容，但 TypeScript 可能会因为参数匹配不严格而报错，或者 React 在传递事件对象时产生混淆。更准确的是，在 JSX 中直接传递函数引用是正确的，但此处报错可能是因为推断的类型不匹配。最稳妥的方式是使用箭头函数封装：`onChange={(val) => handleWorkflowChange(val)}`。
2. **逻辑问题**: 用户切换工作流时，`selectedAlgoId` 虽然在 `handleWorkflowChange` 内部被置空了，但由于它是受控组件（`value={selectedAlgoId}`），且状态更新是异步的，最好显式确保重置逻辑清晰。
3. **文件命名**: 用户希望预测结果文件名的各部分（除了原始文件名保留）都尽量英文/拼音化，避免中文乱码或不规范。

**修复方案**:
1. **修复 `onChange` 类型**: 将 `onChange={handleWorkflowChange}` 改为 `onChange={(val) => handleWorkflowChange(val)}`。
2. **重置模型选择**: 已经在 `handleWorkflowChange` 中通过 `if (!restoreAlgoId) setSelectedAlgoId(null);` 实现了。这部分逻辑是正确的，只要确保 `handleWorkflowChange` 被正确调用即可。
3. **文件名全英文**:
    *   在 `ml_service.py` 中，已经对 `algo_tag` 进行了去除非字母数字字符的处理。
    *   对于原始文件名 `original_name`，如果包含中文，直接截取可能会有编码问题或仍保留中文。建议使用 `pinyin` 库（如果不想引入新库，可以使用简单的字符过滤或哈希）或者保留原始文件名（用户说“训练文件的文件名字还是保持全部比较好”，这里指的可能是上传的原始文件保留中文，但预测结果文件名希望是英文）。
    *   **修正理解**: 用户说“文件命名尽量都给弄成英文的”，指的是生成的预测文件名。但又说“训练文件的文件名字还是保持全部比较好”。这可能意味着：生成的 `pred_` 文件名中，除了保留原始文件名的一部分（哪怕是中文），其他部分（如模型名、时间戳）要是英文。
    *   **后端修改**: 确保模型名（如“逻辑回归”）被转换为英文缩写（如 `LogisticRegression` 或 `LR`）。目前前端传的是 `label`（中文），需要在前端做一个映射，或者后端处理。最简单的是前端传 `Algorithm Name` 的英文版，或者后端维护一个映射表。鉴于 `ml_service.py` 里已有算法列表，可以在后端加一个 `CN_TO_EN` 的映射。

**具体修改**:

1.  **Frontend (`Prediction.tsx`)**:
    *   修改 `Select` 的 `onChange` 调用方式。
    *   确认切换工作流时 `selectedAlgoId` 被置 `null`。

2.  **Backend (`ml_service.py`)**:
    *   添加一个 `ALGO_NAME_MAP`，将中文算法名映射为英文简写（如 "逻辑回归" -> "LogisticRegression"）。
    *   在生成文件名时，使用映射后的英文名。

**代码变更预览**:

**Frontend**:
```tsx
<Select 
    onChange={(val) => handleWorkflowChange(val)}
    // ...
>
```

**Backend**:
```python
ALGO_NAME_MAP = {
    "逻辑回归": "LogisticRegression",
    "决策树": "DecisionTree",
    # ...
}
# ...
algo_tag = ALGO_NAME_MAP.get(algorithm_label, algorithm_label)
# ...
```
