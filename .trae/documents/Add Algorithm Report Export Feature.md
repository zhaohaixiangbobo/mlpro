# 新增算法报告导出功能

我将实现一个支持 Word (docx) 和 PDF 格式的可视化报告导出功能，并将操作入口放置在可视化结果页面的指定位置。

## 技术实现方案

### 1. 后端 (Python)
- **新增路由**: `backend/app/api/report.py`
- **接口定义**: `POST /api/report/export`
- **参数**: `workflow_name` (工作流名称), `format` (导出格式: "docx" 或 "pdf")
- **核心逻辑**:
    1.  **加载工作流**: 读取当前用户指定的工作流 JSON 配置文件。
    2.  **提取数据**:
        -   **EDA 数据**: 读取关联的 `.meta.json` 文件获取数据统计信息（列类型、缺失值、异常值等）。
        -   **算法指标**: 遍历工作流节点，提取所有已运行模型的评估结果。
    3.  **生成图表**:
        -   使用 `matplotlib` 动态生成 **雷达图** (分类对比)、**柱状图** (回归误差) 和 **散点图** (PCA)。
    4.  **生成文档**:
        -   使用 `python-docx` 和 `reportlab` 分别生成 Word 和 PDF 文档。
        -   **报告结构**:
            -   **一级标题**: 算法分析报告
            -   **二级标题**: 1. 输入数据探索性分析 (EDA) [含表格]
            -   **二级标题**: 2. 算法可视化结果分析 [含图表和表格]
- **集成**: 在 `backend/app/main.py` 中注册新路由。

### 2. 前端 (React)
- **API 服务**: 更新 `frontend/src/services/api.ts` 添加 `exportReport` 下载接口。
- **界面更新 (`Dashboard.tsx`)**:
    -   **位置调整**: 在页面顶部的标题栏区域，**"当前工作流" 下拉选择框的左侧** 添加 **"导出报告"** 按钮。
    -   **交互**: 点击按钮弹出模态框，允许用户选择导出格式 (Word / PDF)，确认后触发下载。

## 验证计划
1.  进入可视化结果页面，选择一个已有结果的工作流。
2.  点击"当前工作流"左侧的"导出报告"按钮。
3.  分别选择 Word 和 PDF 格式进行导出。
4.  检查导出文件中是否包含 EDA 表格、算法对比图表及详细指标数据。
