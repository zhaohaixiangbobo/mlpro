# 数据处理与参数配置增强计划

## 1. 前端架构升级

### 1.1 新增属性面板组件 (`PropertyPanel`)

* **位置**: 画布右侧固定区域 (Sider, width 300-400px)。

* **触发**: 点击节点参数部分时显示，点击画布空白处隐藏，避免拖拽时候误触。

* **功能**: 根据当前选中节点的类型 (`category` / `label`) 渲染不同的配置表单。

* **数据流**:

  * 接收 `selectedNode` 信息。

  * 通过 `Workflow.tsx` 传递的 `onNodeDataChange` 回调实时更新节点数据 (`data.params`)。

  * 自动获取上游节点的元数据（如列名列表）用于下拉选择。

### 1.2 专用参数表单开发

* **缺失值处理表单**:

  * 方法选择: 删除行 (drop), 均值 (mean), 中位数 (median), 众数 (mode)。

  * 列选择: 多选下拉框 (支持全选/反选)。

* **标准化处理表单**:

  * 方法选择: Z-score (standard), Min-Max (minmax)。

  * 列选择: 仅显示数值型列。

* **模型参数表单**:

  * 针对不同算法显示特定参数 (如 随机森林的 `n_estimators`, 决策树的 `max_depth`)。

### 1.3 节点状态与错误反馈

* 增强 `AlgoNode` 组件：

  * 新增 `failed` 状态样式 (红色边框 + 错误图标)。

  * 悬停错误图标时显示 Tooltip: 包含错误类型、步骤和建议。

## 2. 后端逻辑增强

### 2.1 API 接口更新

* 更新 `POST /api/workflow/run` 接口，支持接收 `preprocessing` 参数列表。

  * 结构示例: `{ ..., preprocessing: [{ method: "missing", params: {...} }, { method: "scale", params: {...} }] }`

### 2.2 机器学习服务 (`MLService`) 升级

* 新增 `preprocess_data` 方法：

  * 实现缺失值处理逻辑 (`SimpleImputer`, `dropna`)。

  * 实现标准化逻辑 (`StandardScaler`, `MinMaxScaler`)。

* 增强 `run_algorithm`：

  * 按顺序执行预处理步骤。

  * 捕获每一步的异常，返回包含 `step` (出错步骤), `message` (错误信息), `suggestion` (建议) 的结构化错误对象。

## 3. 交互与集成

* **Workflow\.tsx**:

  * 维护 `selectedNode` 状态。

  * 实现 `findUpstreamNode` 逻辑，用于向属性面板传递上游数据（主要为了获取列名）。

  * 在 `handleRun` 中构建包含预处理参数的完整 Payload。

## 4. 验证计划

* 测试缺失值填充功能（选择特定列填充均值，验证运行成功）。

* 测试标准化功能（验证数值范围变化）。

* 模拟错误场景（如对非数值列进行均值填充），验证错误提示是否准确显示在节点上。
